# ReLU.chat — Development Workspace

This directory contains development-only assets used for training, testing, and
auditing the ReLU.chat chatbot engine.

## Directory Layout

| Directory | Purpose |
|-----------|---------|
| `audits/` | Audit reports, test plans, validation templates |
| `benchmarks/` | Performance benchmarks (latency, memory, throughput) |
| `datasets/` | Raw training datasets and prompt collections |
| `exports/` | Model exports, annotated KB dumps, analysis artifacts |
| `scripts/` | Development tools (training, annotation, testing) |

## Scripts

### train-policy.py
RL training pipeline for the WASM policy engine. Generates policy.wasm +
policy.weights.bin + updated policy.manifest.json.

```bash
pip install -r scripts/requirements.txt
python scripts/train-policy.py --bot game-theory --epochs 1000
```

### annotate-fragments.py
Annotates a knowledge-base.js file with fragment-level metadata
(truth_confidence, difficulty, style, etc.) for the policy engine.

```bash
python scripts/annotate-fragments.py ../chat/game-theory-chat/js/knowledge-base.js
```

### test-policy-runtime.js
Node.js test suite for the policy runtime (no browser/WASM needed).

```bash
node scripts/test-policy-runtime.js
```

## Notes
- Files in `/dev` are for local development only and are NOT deployed to
  production via FTP. They ARE pushed to the public GitHub repo as part of
  the open-source project.
- Training artifacts (`.onnx`, `.wasm`, `.bin`) are generated by the scripts
  and placed into `/assets/models/policy/`.
- See `scripts/requirements.txt` for Python dependencies.
